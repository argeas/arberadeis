Execution Logic

Scan Phase (every 500ms)
1. Poll all active venues for market prices
2. For each market: calculate YES_ask + NO_ask
3. Intra-platform: If total < $1.00 on same venue → opportunity
4. Cross-venue: Find cheapest YES + cheapest NO across venues
5. Deduct fees per venue side → net spread
6. If net spread ≥ min threshold → trigger execution
Execute Phase
1. Size position: min(max_size, yes_liquidity, no_liquidity)
2. Order legs by venue speed: Jupiter → Polymarket → Kalshi
3. Leg 1: Place order on fastest venue
4. Leg 2: Immediately place on second venue
5. Both fill → log P&L, update portfolio
6. Leg 2 fails → ORPHAN → attempt market exit on leg 1
Kill Switches
• Daily loss ≥ limit → halt all trading
• Orphan losses ≥ budget → stop new arbs
• >3 open orphans → halt
